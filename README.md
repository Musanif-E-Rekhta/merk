# merk

Axum + SurrealDB backend for the Musanif Urdu reading platform. Exposes a GraphQL API at `/api/graphql` covering books, chapters, annotations, reviews, translations, collections, bookmarks, profile, and the admin ingestion pipeline. REST is reserved for the three places where GraphQL doesn't fit: multipart upload, binary page previews, and operational probes.

The server composes a set of sibling crates extracted out of merk so they can be reused or replaced:

| Crate | Responsibility |
|-------|----------------|
| [`merk-auth`](../merk-auth) | Argon2id + HS256 JWT |
| [`merk-totp`](../merk-totp) | TOTP + AES-GCM secret encryption |
| [`merk-rbac`](../merk-rbac) | role/permission `Authorizer` + `RbacStore` trait |
| [`merk-blob-store`](../merk-blob-store) | `BlobStore` trait + Local/GCS/S3 adapters |
| [`merk-events`](../merk-events) | broadcast bus + outbox trait |
| [`merk-ingest`](../merk-ingest) | pipeline orchestrator + `Step` trait |
| [`merk-observability`](../merk-observability) | tracing + Prometheus + OTLP |
| [`merk-axum-middleware`](../merk-axum-middleware) | `BearerAuth` extractor |
| [`merk-migrations`](../merk-migrations) | SurrealDB schema migrator |
| [`merk-seed`](../merk-seed) | dev seed runner (separate workspace) |
| [`musanif-contracts`](../musanif-contracts) | shared types + `schema.graphql` |

## Quickstart

```bash
# 1. SurrealDB + Prometheus + Grafana
docker-compose up -d

# 2. Migrations + 7 personas + Sarab fixtures
MERK_ENV=local cargo run --manifest-path ../merk-seed/Cargo.toml -- --reset

# 3. Run merk
cargo run                           # binds 0.0.0.0:9678 by default

# 4. Refresh the GraphQL schema for the frontend's cynic codegen
cargo run --bin merk-dump-schema    # → ../musanif-contracts/schema.graphql

# 5. Run integration tests (in-memory SurrealDB; no docker needed)
cargo test --test auth_flow
```

The startup banner prints the active bind address, the GraphQL endpoint, and the Scalar OpenAPI docs URL.

## Configuration

Loaded by [`envy`](https://github.com/softprops/envy) into a typed `AppConfig`. Defaults are appropriate for local dev — set `JWT_SECRET` explicitly in production (the dev default is rejected when built with `--release`).

| Variable | Default | Description |
|----------|---------|-------------|
| `HOST` | `127.0.0.1` | Network interface |
| `PORT` | `9678` (HTTP) / `8443` (TLS) | Listen port |
| `ENABLE_TLS` | `false` | Toggle auto-generated rcgen self-signed cert |
| `TLS_ALT_NAME` | *(empty)* | Extra SAN on the generated cert |
| `SURREALDB_URL` | `ws://127.0.0.1:8000` | SurrealDB connection URL |
| `SURREALDB_USER` | `root` | SurrealDB root username |
| `SURREALDB_PASS` | `root` | SurrealDB root password |
| `SURREALDB_NS` | `merk` | Namespace |
| `SURREALDB_DB` | `merk` | Database |
| `JWT_SECRET` | *(dev default)* | HS256 signing secret — must be ≥ 32 chars |
| `MAIL_TRANSPORT` | `log` | Outbound mail backend (`log`, `noop`; `smtp` planned) |
| `MAIL_RESET_URL_TEMPLATE` | `https://app/reset?token={token}` | Templated reset URL |
| `MAIL_VERIFY_URL_TEMPLATE` | `https://app/verify?token={token}` | Templated verify URL |
| `CORS_ORIGINS` | *(empty → permissive)* | Comma-separated allowlist |
| `RUST_LOG` | `merk=debug,tower_http=debug,axum=info,info` | Tracing filter |

## GraphQL

Schema source of truth: [`musanif-contracts/schema.graphql`](../musanif-contracts/schema.graphql), regenerated by `cargo run --bin merk-dump-schema`. The frontend uses cynic codegen against that file — when you add a resolver, dump the schema so the contract crate stays in sync. CI is meant to enforce `git diff --exit-code schema.graphql` after a fresh dump.

Top-level queries (non-exhaustive):
- `me`, `myStats`, `myReadingSessions`, `myFollowing`, `myHighlights`, `myCollections`, `myBookmarks`, `myReadingGoal`, `myContinue`, `my2FaStatus`
- `books`, `book`, `featured`, `booksByAuthor`, `chapters`, `chapter`
- `authors`, `author`
- `bookReviews`, `chapterReviews`, `chapterHighlights`, `chapterComments`, `wordTranslations`
- Admin: `ingestionJobs`, `ingestionJob`, `jobSteps`, `jobLog`, `chapterDrafts`, `coverVariants`, `publishChecks`, `adminUsage`

Top-level mutations (non-exhaustive):
- Auth: `registerUser`, `loginUser`, `login2FaComplete`, `refreshToken`, `logoutUser`, `forgotPassword`, `resetPasswordWithToken`, `requestEmailVerification`, `verifyEmail`
- 2FA: `setup2Fa`, `verify2Fa`, `disable2Fa`
- Profile: `updateProfile`, `changePassword`, `deleteMe`
- Reader: `upsertBookmark`, `upsertReadingGoal`, `recordReadingSession`, `createBookReview`, `createHighlight`, `createComment`, `voteComment`, `submitTranslation`, `voteTranslation`, …
- Admin: `createIngestionJob`, `startIngestionJob`, `approveChapterDraft`, `flagChapterDraft`, `selectCoverVariant`, `publishIngestionJob`, `updateAdminBook`

Subscriptions (over `graphql-transport-ws` at `/api/graphql/ws`): `onPipelineStep`, `onPipelineLog`, `onPipelineCompleted`, `onChapterDraft`.

## Database

SurrealDB schema lives in [`merk-migrations`](../merk-migrations). Migrations are embedded into the merk binary at compile time and applied at startup; the `_migrations` table tracks what's been applied. `cargo run --manifest-path ../merk-seed/Cargo.toml -- --reset` runs migrations to head and seeds 7 personas + Sarab pipeline fixtures.

Highlights worth knowing:
- RBAC: `user –[assigned_role]→ role –[has_permission]→ permission`, traversed natively.
- Reviews/votes/translations: SurrealDB events maintain denormalised aggregates (`avg_rating`, `helpful_count`, `upvotes`) so reads don't aggregate.
- Word translations resolved chapter → book → global.
- Vector search: chapter chunks are indexed via HNSW (cosine, 1536-dim) for semantic lookup.
- Refresh tokens: stored hashed (SHA-256), rotated on every use, revocable via `logoutUser`.

See [`docs/schema_design.md`](docs/schema_design.md) for the full entity map, ERD, GraphQL schema, and HTTP examples.

## Authentication

- Tokens are HS256 JWTs signed with `JWT_SECRET`. Access tokens expire in **15 minutes**; refresh tokens (opaque, SHA-256 hashed at rest) live for **30 days** and rotate on every use.
- Bearer auth: pass `Authorization: Bearer <access_token>` on protected operations.
- 2FA flow: `loginUser` returns `requires2Fa: true` + an opaque `challenge`; the client passes the challenge plus a TOTP code (or recovery code) to `login2FaComplete` to mint the real session.
- Logout: `logoutUser(refreshToken)` revokes the refresh-token row server-side.

## Tests

```bash
cargo test --test auth_flow         # auth happy/edge paths against in-memory SurrealDB
```

Seven scenarios pass today: register → login, wrong-password / wrong-email, change-password, refresh-token rotation + revocation, logout, forgot-password mail dispatch.
